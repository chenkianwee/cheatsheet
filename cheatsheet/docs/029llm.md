## Post training and tuning (alignment)
- Fine tuning is a key method of post training. It is part of the post training process
    - post training is often times done by model developers to optimize the model 
    - app developers can further do fine tuning after post training by model developers to optimize the model for domain specific task.

## Retrieval Augmented Generation (RAG)
- What are the benefits of Retrieval-Augmented Generation? [Link](https://aws.amazon.com/what-is/retrieval-augmented-generation/)
    - With RAG, an information retrieval component is introduced that utilizes the user input to first pull information from a new data source. The user query and the relevant information are both given to the LLM. The LLM uses the new knowledge and its training data to create better responses.
- The Best RAG Frameworks in 2026: A Comprehensive Guide to Building Superior Retrieval-Augmented Generation Systems [Link](https://martinuke0.github.io/posts/2026-01-06-the-best-rag-frameworks-in-2026-a-comprehensive-guide-to-building-superior-retrieval-augmented-generation-systems/#top-5-best-rag-frameworks-in-2026)

## Practical implementation
1. Jan (higher level model management GUI)
    - make sure CUDA is installed with your NVIDIA graphic card
    - for ubuntu install jan using the .deb instead of the flatpak, flatpak might not be able to detect your gpu
    - sudo apt install libnccl2 libnccl-dev
    - to run gemma-4-e4b need to turn off mtp and set the gpu level to 30 for optimum performance
2. llama.cpp (lower level run time)
3. copilot harness
4. RAG for more reliable retrieval of information
5. develop agentic skills and workflow
6. connect agents to MCP to extend functionalities
7. fine tuning if required
